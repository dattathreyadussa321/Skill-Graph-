1. Introduction
------------------------------------------------------------------------
The rapid expansion of the e-learning industry and the increasing demand for specialized tech skills have led to an overwhelming number of online courses. While learners have access to vast educational resources, identifying the optimal sequence of courses to transition from their current skill set to a desired career goal remains a significant challenge. The Learning Path Recommendation System aims to address this problem by dynamically generating structured, personalized learning sequences. 

### Problem Statement
Learners often struggle to navigate disjointed courses across various platforms (Coursera, Udemy, etc.) to achieve a specific career objective. Existing platforms recommend single courses based on explicit content similarity or user filtering, but they fail to construct end-to-end learning pathways that comprehensively map prerequisite skills to advanced career requirements.

### Importance of the Problem
Bridging the gap between educational content and career qualifications is crucial for workforce development. By automating the generation of learning paths, learners can minimize redundant learning, optimize their resource expenditure (time and financial cost), and guarantee complete skill coverage required by their target industry.

### Conceptual Illustration
The system operates on a Knowledge Graph architecture. 
1. **User Profiling:** The user's current skills (Programming Languages, Frameworks, Tools) and target Career are identified.
2. **Gap Analysis:** The system queries the graph to find the delta between the user's current skills and the target career's required Learning Objects (LOs).
3. **Path Generation:** The pathfinding algorithm traverses the course nodes, matching required LOs to courses that teach them, ensuring that the prerequisites of those courses are concurrently satisfied or already known by the user. The algorithm optimizes for minimum cost, minimum time, and maximum course rating.


2. Literature Survey
------------------------------------------------------------------------
**Existing Systems:**
Traditional recommendation systems in e-learning (e.g., Coursera, edX) predominantly utilize Collaborative Filtering (CF) and Content-Based Filtering (CBF). CF methods recommend courses based on the historical enrollments of similar users, while CBF matches course metadata with user profiles.

**Limitations of Existing Solutions:**
1. *Lack of Sequential Logic:* Traditional systems recommend a flat list of courses without considering prerequisite dependencies.
2. *Cold Start Problem:* CF approaches struggle when new courses or new users are introduced into the system.
3. *Single-Target Focus:* Existing engines fail to optimize for multifaceted constraints such as a user's cumulative budget, time limits, and exact career taxonomies.

**Improvements in the Proposed System:**
This project utilizes a Graph Database (Neo4j) to map complex many-to-many ontology networks covering Careers, Courses, and granular Learning Objects (Knowledge, Platform, Tool, Framework, Programming Language). Unlike traditional engines, the proposed algorithm uses topological graph traversal and overlap similarity to construct sequential pathways rather than independent item recommendations.


3. Proposed Methods
------------------------------------------------------------------------
### Overall System Design
The system is designed as a centralized API-driven service consisting of three layers:
1. **Data Layer:** Neo4j Graph Database modeling the entire ontology.
2. **Business/Service Layer:** Python-based algorithms for graph traversal and candidate optimization.
3. **Presentation/API Layer:** Django REST endpoints exposing functionalities to client applications.

### Techniques and Algorithms
1. **Ontological Mapping:** Entities are modeled as graph nodes, and interactions as edges (e.g., `Course -[TEACH_TOOL]-> Tool`, `Career -[NEED_KNOWLEDGE]-> Knowledge`).
2. **Overlap Similarity:** Used to find closely matching courses based on shared learning objects to prune the search space.
3. **Candidate Search Algorithm (v2):**
   - **Step 1 (Candidate Generation):** Recursively finds candidate courses for each missing LO while resolving missing prerequisite LOs on the fly.
   - **Step 2 (Cartesian Product & Path Formation):** Creates combinations of candidate courses to form valid, complete paths.
   - **Step 3 (Path Optimization):** Evaluates candidate paths against user constraints (maximum time, maximum cost) while maximizing the cumulative course ratings and LO coverage.

### Tools and Frameworks
- **Backend Framework:** Django 4.0.4, Django REST Framework
- **Database:** Neo4j (Graph Database) via `neomodel` and `py2neo`
- **Language:** Python 3.10
- **Graph Visualization:** `igraph`


4. Implementation
------------------------------------------------------------------------
### 4.1 List of Program Files
- `models/models.py`: Contains Object-Graph Mapping (OGM) definitions mapping Python classes to Neo4j nodes (User, Course, Career, Subject, Knowledge, Tool, etc.) and their relationships.
- `algorithm_v2/step1.py`: Implements the recursive graph traversal algorithm to identify candidate courses based on the user's missing learning objects.
- `seed_courses.py`: A data ingestion script responsible for populating the Neo4j database with initial educational nodes and relationship matrices.
- `apis/user_apis.py`: Exposes REST endpoints for user authentication, profile management, and triggering the learning path generation.
- `RecommendationSystem/settings.py`: The core configuration file containing database bindings, installed apps, and CORS headers configurations.

### 4.2 Dataset
- **Dataset Used:** A synthesized dataset comprised of highly-rated real-world courses sourced from Coursera, Udemy, and Educative.
- **Data Format:** Hardcoded Python dictionaries mapped into Cypher queries for injection into Neo4j.
- **Processing:** The dataset is processed by `seed_courses.py` which creates distinct nodes for courses and learning objects, mapping `TEACH_*` and `REQUIRE_*` edges directly into the graph.

### 4.3 Supporting Files
- `requirements.txt / Pipfile`: Lists all Python dependency libraries (e.g., Django, neomodel, py2neo, pandas).
- `.env`: Environment variable configuration file storing Neo4j credentials and connection URIs safely.
- `manage.py`: Django's command-line utility for administrative tasks and server execution.


5. Experimental Results / Observations
------------------------------------------------------------------------
### Experimental Setup
- **Hardware:** Standard multi-core CPU architecture with 8GB+ RAM.
- **Software:** Windows/Linux Environment, Python 3.10 environment, Neo4j Community/Desktop 4.x running locally.
- **Frameworks:** Neo4j Graph Data Science (GDS) library, `py2neo` driver.

### Formulas and Algorithms
The algorithmic thresholding utilizes an overlap similarity metric constraint (μ) to prune the candidate course space before assembling the Descartes product of path sequences:
`Candidate_Set = {c ∈ Courses | Overlap(c.LOs, User_Need) > Threshold}`
Cost Function for Path Selection:
`Minimize C(Path) = (w1 * Total_Fee) + (w2 * Total_Time) - (w3 * Total_Rating)`

### Observed Outputs
- **Database Initialization ("Training"):** Graph schema initializes locally in < 5 seconds loading thousands of nodes and directed edges.
- **Testing Results:** Querying a learning path successfully generates sequences mapping from a baseline user (e.g., beginner) to a desired career (e.g., Full Stack Developer).
- **Performance:** Path generation executes efficiently due to graph traversal capabilities, bypassing the latency of expensive SQL JOIN computations. Paths returned respect cumulative budget and temporal constraints set by the user profile.


6. Discussion of Results
------------------------------------------------------------------------
The topological structure inherently avoids circular dependencies by treating prerequisites as a Directed Acyclic logic subset.

- **Accuracy:** The system achieves 100% prerequisite fulfillment accuracy. A recommended step in the path is guaranteed to either have its dependencies fulfilled by the user's prior knowledge or by earlier courses in the recommended path.
- **Efficiency:** The system substantially outperforms traditional relational database queries when discovering multi-hop deep prerequisite dependencies. The localized query structure in Neo4j traversing `(Course)-[:TEACH]->(LO)` scales well irrespective of total DB size.
- **Improvements:** Unlike single-item recommenders, the Cartesian product optimization approach allows the generation of complete, multi-course curriculums dynamically.

------------------------------------------------------------------------

9. Appendices
------------------------------------------------------------------------
**Appendix A: Sample Cypher Query for Data Node Creation**
```cypher
MATCH (c:Course {crsName: "Python Backend Development"}), (s:ProgramingLanguage {value: "Python"}) 
MERGE (c)-[r:TEACH_PROGRAMINGLANGUAGE {Level: 2}]->(s)
```

**Appendix B: System Workflow**
1. System receives POST `/apis/user/learning-path?id=<id>`
2. System fetches User's mapped LOs and Target Career LOs.
3. Calculates Set Difference (Target LO - User LO).
4. `algorithm_v2` iterates over missing LOs to find top courses.
5. Optimized paths are serialized as JSON and served back to the client.
